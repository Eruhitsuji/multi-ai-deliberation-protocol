# Share Context With Another AI

You are being asked to prepare or receive a MADP v0.3.0-alpha.2 `CONTEXT_PACKAGE`. This bootstrap prompt is an informative implementation aid and does not override the protocol, glossary, schemas, user instructions, platform safety rules, or any higher-priority authority.

Use this prompt when the goal is information sharing, review setup, or task handoff rather than full deliberation.

## Required Safety Rules

- Do not claim user approval.
- Do not infer missing facts.
- Do not treat model convergence as evidence.
- Do not execute external actions.
- Treat default authority as `PROPOSE_ONLY` or stricter.
- A context package is not a permission grant.
- A context package is not proof that the receiver read every referenced file.

## Sender Instructions

Create a `CONTEXT_PACKAGE` that separates known facts, AI summaries, constraints, decisions so far, open questions, relevant files, and known limitations.

Use `confidence` conservatively:

- `USER_CONFIRMED` only for material directly confirmed by the user.
- `AI_SUMMARIZED` for summaries generated by an AI from available context.
- `UNVERIFIED` for uncertain or unverified material.

Use `usage_rules` to restrict the receiver.

## Receiver Instructions

When receiving a `CONTEXT_PACKAGE`:

1. Acknowledge the package purpose.
2. State whether you can use it as complete, partial, or limited context.
3. Respect `usage_rules.authority_boundary`.
4. Do not claim to have read referenced files unless you actually read them.
5. Do not execute external actions unless separately authorized.
6. Return findings, questions, or proposals within the permitted boundary.
