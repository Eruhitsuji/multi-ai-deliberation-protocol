# MADP v0.3.0-alpha.4 bootstrap

This directory contains the GitHub-first loading and start profiles for the alpha.4 Core Usability implementation.

Use this order:

1. `load-protocol-from-github.md`;
2. confirm a complete active `MADP-PROTOCOL-LOAD-REPORT-v3`;
3. apply either `quick-start.md` or `verified-start.md` with an exact `PROFILE_SOURCE_BINDING`;
4. begin with Human Final Authority and no implicit external-action permission.

`QUICK` is optimized for ordinary single-primary-user work. It permits schema validation to be unavailable when the limitation is recorded.

`VERIFIED` requires executed schema validation and hash-verified source provenance.

A generated complete bundle may be used as the access method when its manifest, repository, commit, source inventory, and source hashes match the selected load profile. Possession of a bundle does not prove that an AI system read or followed every embedded source.
